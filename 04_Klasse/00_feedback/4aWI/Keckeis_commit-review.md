**Commit `b7f1c21` — Finished design Patterns**

This is the most substantial commit. Introduces 4 design patterns cleanly in one focused commit.

**Strategy Pattern** (via `Movement`, `RightMovement`, `LeftMovement` + actor `setMovement()`):
- `Movement` interface with `x`, `y`, `speed` and `update(deltaTime)`.
- `RightMovement` and `LeftMovement` implement it correctly (x += speed * deltaTime).
- `Actor` abstract class holds `movement: Movement` and delegates `update()` to it. Runtime swap via `setMovement()`. Correct and clean.

**Observer Pattern** (`ActorObserver`, `BoundaryBounceObserver`):
- `ActorObserver` interface with `onBoundaryHit(actor, side)`.
- `BoundaryBounceObserver` reacts by swapping the movement strategy — an elegant combination of Observer + Strategy.
- `Actor` base class maintains an observers list with `addObserver`, `removeObserver`, `notifyBoundaryHit`. `MyGame.update()` calls `actor.notifyBoundaryHit("right"/"left")` on boundary collision.
- This is excellent — the Observer decouples collision detection from movement direction change.

**Factory Pattern** (`ActorFactory`):
- `ActorFactory` with `createCircle()`, `createRectangle()`, `createSuperCircle()` methods and `createDemoActors()` convenience method.
- Note: `ActorFactory` is also a Singleton (see below) — the Factory and Singleton patterns are combined in one class, which is a valid real-world approach.

**Singleton Pattern** (`ActorFactory.getInstance()`):
- Private constructor, static `instance`, `getInstance()` — textbook correct Singleton implementation.
- `MyGame.init()` uses `ActorFactory.getInstance()` — the Singleton is actively used, not just declared.
